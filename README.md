# dynamic-bot-weibo

微博动态源插件，用于让 dynamic-bot 订阅微博用户的新动态，并支持解析微博链接。

## 功能

- 使用微博 Cookie 登录。
- 查询微博用户资料。
- 关注和取关微博用户。
- 可选关注分组。
- 检测订阅用户的新微博。
- 解析微博正文和用户链接。
- 自动补全长文内容。
- 登录失效时暂停轮询，避免反复请求。

当前版本暂不支持用户名搜索。添加订阅时建议使用微博 UID。

## 安装

构建插件：

```powershell
.\gradlew.bat fatJar --offline
```

把生成的 `*-all.jar` 放入主程序的：

```text
plugins/
```

然后启动或重启 dynamic-bot，在后台启用插件。

## 登录

微博插件使用 Cookie 登录。可以在后台配置中填写 Cookie。

Cookie 失效后，插件会检测到未登录状态，并暂停轮询。更新 Cookie 后可以恢复。

## 常用配置

微博接口比较敏感，所以轮询默认关闭。需要推送微博动态时再开启。

- `启用轮询`：是否定时检测新微博，默认关闭。
- `轮询间隔`：多久检查一次新微博，默认 60 秒。
- `请求间隔`：连续请求微博接口之间等待多久，默认 1 秒。
- `补发时间窗口`：启动时是否补发最近微博，默认 0，不补发。
- `关注分组名`：关注后加入哪个微博分组。
- `自动创建关注分组`：分组不存在时是否自动创建。
- `未登录暂停阈值`：连续未登录几次后暂停轮询。

## 订阅

添加微博订阅时使用微博 UID。订阅后，如果开启了主程序的自动关注，插件会尝试关注该用户。

如果配置了关注分组，关注成功后会尝试加入该分组。

## 动态检测

插件主要通过微博关注流检测新动态。这样可以避免每个订阅用户都单独请求一次，减少风控风险。

首次启动默认不会推送旧微博。如果需要补发最近一段时间的内容，可以设置补发时间窗口。

## 使用建议

- 不需要微博推送时保持轮询关闭。
- 不要把轮询间隔设得过短。
- Cookie 失效后及时更新。
- 如果出现风控或请求失败，先暂停轮询或增大请求间隔。
